◆管理表xlsを扱う際の注意点

 

１、管理表を編集する前に一度xlsを立ち上げ、

ツール→オプション→編集タブの”リンクの自動更新前にメッセージを表示する”

チェックボックスをOffに設定してください

 

　　→Offにしないと、毎回確認ダイアログが表示されます

 

２、管理表xlsをドラック＆ドロップで他のフォルダに移動させないでください

 

→Demo3Dのシーン管理表のxlsは「外部リンク」という機能を使い、複数の管理表で

ファイルの設定データを共有しています。

 

ドラッグ＆ドロップでファイルを移動させてしまうと、このリンクが切れてしまいますので、

移動やコピーを行いたい場合は、エクセルファイルを開き、ファイルメニューから

“名前をつけて保存”を用いてください

 

 

３、たとえ値が同じでも、セルを異なる項目のセルにコピーしないでください

 

→各セルには最小値/最大値などのチェックのために各種入力制限をかけています。

　異なる項目のセルをコピーしてしまうと、この入力規則まで上書きされてしまうため

　各セルが意図した動作をしなくなってしまいます

 

 

◇外部リンクが切れた場合の復旧方法

　　うっかりリンクが切れてしまった場合、

　　「このブックには更新できないリンクが１つ以上含まれています」というダイアログが出ます。

 

　　まず、”継続”をクリックし、そのままブックを開きます。

　　次に、<command>と<value>のシート保護をパスワード“demo3d”で解除してください。

　　　＊シートの保護を解除するには、エクセルメニュー→ツール→保護→シート保護の解除

 

　　ファイルメニューの編集→リンクの設定を選びます。

　　「リンクの編集」という名前のウィンドウが出るので、“リンク元の変更”をクリックしてください。

 

　　すると、ファイルを選ぶエクスプローラーが表示されるので、

　　pokemon_wb/resource/demo3d/conv/demo3d_cmd.xls を開いてください。

　　最後に、もう一度<command>と<value>のシートに保護をかけて、セーブしてください。

　　　＊シートを保護するには、エクセルメニュー→ツール→保護→シートの保護

 

 

◆用語説明

 

◇ユニット

一つのimdと、それと関連付けられたアニメファイルのセットを「ユニット」と呼んでいます。

demo3dアプリでは、シーンに配置する3Dのリソースをユニット単位で指定できます。

 

また、ユニットは　タイムゾーン、プレイヤーの性別、シーンIDに

よって読み替える指定ができます。

 

◇シーンID

デモの呼び出し側がデモに対して与えるナンバーです。たとえば、ソフトウェアバージョンや

観覧車デモのパターンを、呼び出し側がデモに対して引渡し、シーンIDによってユニットを

読み替えるなどのことが可能です。

 

◆Tips

 

◇demo3d で2画面表示をする場合の、カメラオフセットの指定について

 

2画面表示をOnにした際の、上下画面それぞれのオフセット値は、

管理表の「2画面連結」の　param1 と param2 で設定することができます。

 

param1 が上画面のオフセット

param2 が下画面のオフセットです

 

デフォルトは0で、param1 と param2 を両方 0に設定すると、上下画面で同じカメラワークになります。

そこから、各シーンに応じて、上下にオフセット値を設定してください

○2画面表示時のカメラ位置調整用

 

　　　・Up/Down キー　  上画面のカメラオフセットを +/- 1 します

　　　・Right/Left キー　下画面のカメラオフセットを +/- 1 します

 

　　　・R / L ボタン　  上下画面のカメラオフセットを同時に +/- 1します。

                         両方同時に動かしたい時はこちらで

 

　　　カメラを動かすと、デバッガーの「出力」ウィンドウに、現在の設定値が表示されます

 

　　　offset def_top = 0.036377, def_bottom = -0.036377

　　　offset_main view = 0.093994, d = 236 o = -9

　　　offset_sub view = 0.002197, d = -140 o= -248

 

　　　d値が現在のオフセット値、 o値が管理表の設定値からデバッグ操作で動かした差分です。

　　　気に入ったカメラーワークを見つけたら、その時のd値の出力を確認し、同じ値を管理表に設定してください

 

　　　ちなみに、上下画面のオフセット値なのですが、単位はドットやMaya上の1unitではありません。

　　　本当にただの小数点値であり、また、アスペクト比が異なるデモ間では、

　　　同じオフセットをいれても同じズレぐあいにはならないのでご注意ください。

 

　一応簡単に説明をいれておきます。読み飛ばしてもOKです。

　　　"1" は 整数部19bit, 少数部12bitの固定小数点値の 1 です。浮動小数点換算で約0.0002くらいですが

　　　この精度の値を小数点値で指定するのは面倒なので、固定少数の直値で指定させています。

 

　　　値がすごく小さいのですが、射影変換行列に渡す値を直に指定しているので、こういう値になります。

◆フィールドOBJ素材の流用

 

　◇概要

　　シャチプロジェクトにおいては、フィールドOBJのモデル素材をdemo3dアプリのユニット素材として

　　使いまわすことができます。完全な共有ではなく、pokemon_wb/resource/fldmmdl(以下fldmmdl)のコンバート時に

　　生成されるビルボードアクター用のimd を、リネーム＆コピーしてdemo3d素材としてアーカイブします。

 

　　また、この構想ではビルボードアクター用のアニメーションファイルも出来る限り共有を図ります。

 

WBにおける全てのフィールドOBJは、管理表resource/fldmmdl/fldmmdl_list.xls で管理されているので、

この表を参考に、流用したいフィールドOBJを決めてください。

 

　◇素材流用設定

　　流用するimd素材は登録式です。

　　Resource/demo3d/fldmmdl_list.txt に、流用したいフィールドOBJリソースのimd名を記述してください。

 

　　リストに記述されたimd が demo3dフォルダのmake時に demo3d/res/fldmmdl フォルダに

プレフィックス“fmdl_”を付加した命名規則でコピーされ更新されます。

    例） オリジナルが hero.imd なら fmdl_hero.imd になります

 

　　fldmmdl_list.txtフォーマット

　　　FLDMMDL_LIST =    hero.imd \　//← “\”マークで行を連結してください

                        heroine.imd \

                        dancer.imd \

                        babygirl2.imd //最後の行に“\”はいりません。つけるとエラーになるので注意！

　　

　　※流用可能なimd は resource/fldmmdl/mmdl_res フォルダに存在するものに限ります。

※新規にリストを追加した際は、増えたimdを必ずsvnに追加することを忘れないでください。

 

　◇フィールドOBJ流用素材フォルダ構成

　　

　　Resource/demo3d/res

　　　-fldmmdl          fldmmdlからコピーしてきたフィールドOBJのimd素材(リストに登録されたものが自動でコピーされます)

　　　-fldmmdl_anm      フィールドOBJのimd間で共有する各種アニメーションファイル(必要に応じて自作します)

 

　◇素材流用の利点

　　Fldmmdl素材の流用にはいくつかの利点があります

 

　　・テクスチャイメージの更新し忘れ予防

 

　　　  デモに取り込む自機のビルボード素材などは、フィールドとデモとで表示されるテクスチャイメージパターンが

同一であることが要求されます。

 

　　　　しかし、resource/fldmmdlフォルダの生成物であるフィールドOBJ素材と、resource/demo3d フォルダで生成されるデモ用素材は別物であり、

　　　　更新＆管理をする人間が異なるため、フィールドOBJの素材更新のたびに、手動でデモ側の素材を追随させるのは非常に大変です。

 

　　　　これを、demo3dリソースのmakeをかけた時に、最新のフィールドOBJ素材をresource/demo3dフォルダにコピーし

　　　　自動で変更を取り込むことで、フィールドOBJ素材の変更追随の手間を削減し、更新忘れを予防します。

 

　　・アニメーションファイルの共有

 

        フィールドOBJ素材は、出力されているマテリアル名、ノード名が全て固定の文字列です。

また、テクスチャイメージ名およびパレット名については、テクスチャサイズや内包するアニメパターンによって

        決定されるタイプ(以下アニメタイプと呼びます)ごとに固定の文字列が割り当てられています。

 

        詳細は下記に示しますが、これらのことによりビルボード素材間でアニメーションファイルの共有が可能です。

 

　◇アニメーションファイルの共有について

 

　　・ica(キャラクターアニメーションファイル)

　　・iva(ビジビリティアニメーションファイル)

        ica 及び iva のアニメ適用情報は imd の ノード番号に依存します。

　　　　このため、全てimd間で共有が可能です。

 

　　・ima(マテリアルカラーアニメーションファイル)

　　・ita(テクスチャSRTアニメーションファイル)

        ima及び itaのアニメ適用情報は imd の マテリアル名<material nameタグ>に依存します。

        フィールドOBJ の imd に出力されるマテリアル名文字列は “obj”で固定にしているため、

　　　　全てのimd間で共有が可能です。

 

        フィールドOBJに適用する ima/ita アニメを作成する時は、アニメ対象のマテリアル名を “obj”として作成してください。

 

　　・itp(テクスチャパターンアニメーションファイル)

        itpのアニメ適用情報は imd の テクスチャ名<tex_image nameタグ>、テクスチャパレット名<tex_palette nameタグ>

及び、マテリアル名<material nameタグ>の3つに依存します。

 

        マテリアル名文字列は “obj”で固定、テクスチャ名とパレット名については、アニメタイプごとに固定の文字列が

        割り当てられるため、同じアニメタイプを持つ imd 間で共有が可能です。

 

        どのフィールドOBJ素材がどのアニメタイプかは、フィールドOBJの管理表resource/fldmmdl/fldmmdl_list.xls に依存します。

<ビルボード><ポケモン>シート内のOBJについては、<モデルサイズ>列と<アニメID>列のパラメータから

アニメタイプが決定されます。<モデル>シート内のOBJは、ビルボードではないためアニメタイプを持ちません。

 

タイプごとのテクスチャ名、パレット名の対応については下図に示します。
