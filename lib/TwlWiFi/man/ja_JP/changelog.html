Note 33-01: (CPS) TCP ソケットが RST を送信しない問題の修正

TCP コネクションにおいて、RST を全く送信しておりませんでした。
これを修正し、接続後の SYN、閉じたポートへの SYN など不適切なパケットに対して RST を返すように修正しました。

Note 33-02: (SOC) TCP の接続先にブロードキャストアドレスが指定できる問題の修正

SOC_Connect 関数で TCP の接続先にブロードキャストアドレスを指定できる問題があり、これを修正しました。

Note 33-03: (TOOL) certconv の修正

ASN.1 の UTF8String タグに対応しました。

SDK Wi-Fi ライブラリ 3.2 RC での既知の問題点

　ありません。

■ TWL-SDK Wi-Fi Version 3.2 PR

(2009/04/16)

Note 32-01: (CPS) CPS_Cleanup 関数の不具合修正

呼び出すタイミングによって CPS_Cleanup 関数が返ってこないという不具合があり、これを修正しました。

Note 32-02: (NHTTP) NHTTP_AddPostDataBinary 関数の返り値の修正

NHTTP_AddPostDataBinary 関数の返り値は成功時に 0 を、失敗時に -1 を返す仕様ですがこれが逆になっており、この不具合を修正しました。

■ TWL-SDK Wi-Fi Version 3.1 PR2

(2009/01/15)

Note 29-01: (CPS) TCP 通信の接続性の向上

状況の悪い通信路において、特定のサーバ実装と TCP 通信を行う際に受信速度が極端に遅くなったり、通信が切断されることがありましたので、これを改善しました。
なお、この変更により、ソケット関連の構造体のサイズが変更されています。
ネットワークを使用するライブラリは再ビルドの必要がありますので、利用しているライブラリの対応状況をご確認ください。

Note 29-02: (SOC/NHTTP/WCM) SOC、NHTTP、WCM の非公開化

SOC、NHTTP、WCM は非公開になりました。
この変更に伴って、関数リファレンスも非公開になっています。
これらの API を直接使用したい方は弊社窓口までお問い合わせ下さい。

Note 29-03: (WCM) 無線デバイスに依存しない WMGameInfo 取得 API の追加

WCM を使用して NintendoZone 情報を取得するために、使用している無線デバイスに因らずに WCM が取得した BssDesc から WMGameInfo を取得する API を新設しました。

SDK Wi-Fi ライブラリ 3.1 PR2 での既知の問題点

　ありません。

■ TWL-SDK Wi-Fi Version 3.1 PR

(2008/12/10)

Note 28-01: (TWL無線) 受信可能な EtherType の制限解除

TWL 無線を使用した場合に IPv4、ARP 以外のパケットはドライバが受信直後に破棄する仕様になっておりましたが、これを辞め WPA 接続に使用する EAPOL 以外の全てのプロトコルを受信し、上位レイヤーに通知するよう変更しました。

Note 28-02: (その他) TWL-WiFi パッケージ内のデモの非公開化

ネットワーク接続を行うアプリケーションは、対戦機能の有無にかかわらず TWL-DWC パッケージに含まれるネットワーク接続関数の使用が必須となっています。
しかし、誤って TWL-WiFi 内のデモ用の接続関数を使用して製品の開発を進めてしまうケースがありましたので、TWL-WiFi のデモを非公開化しました。

SDK Wi-Fi ライブラリ 3.1 PR での既知の問題点

　ありません。

■ TWL-SDK Wi-Fi Version 3.0 正式版

(2008/10/23)

Note 27-01: (NSSL/NHTTP) NSSL ライブラリの非公開化

NSSL ライブラリを非公開にしました。
非公開化に伴い NHTTP ライブラリが使用する SSL は CPS の SSL になります。

この変更は 3.0RC3 で入っていましたが、changelog から漏れていました。

Note 27-02: (WCM) TWL 無線関連のコードを TWL 拡張領域に配置するように修正

WCM の TWL 無線に関連するコードが NITRO でも利用可能なメインメモリに配置されておりましたので、HYBRID アプリ開発の支障にならないよう TWL 拡張メモリ領域に配置されるように変更しました。

Note 27-03: (WCM) WCM が必要とするワークバッファのサイズ削減

TWL 無線を使用する場合に WCM が必要とするワークバッファのサイズを 64KB から 18KB に削減しました。

SDK Wi-Fi ライブラリ 3.0 正式版 での既知の問題点

　ありません。
