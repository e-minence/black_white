DWC_SendUnreliable

構文

#include <dwc.h>

BOOL DWC_SendUnreliable( u8 aid, const void* buffer, int size );

説明

マッチメイク完了後、aidで指定された相手にUnreliable送信を行います。
Unreliable送信はUDPそのもので、送信したパケットが欠落したり、順番が入れ替わったりする可能性があり、このような場合でもエラーを返しません。 その代わりにデータの到着確認も再送も行わないのでReliable送信より高速です。

ひとつのパケットで送信できる最大のサイズ（デフォルト1465Byte）が決まっているため、Unreliable送信ではこのサイズ以上のデータを送信することはできません。（Reliable送信の場合は分割送信されます。）
なお、この最大データサイズはDWC_SetSendSplitMax関数で変更することができますが、様々な設定の通信機器に対応するためには、デフォルトのサイズより大きい値には設定しないでください。

DWC_SetUserSendCallback関数で送信完了コールバックを設定していた場合はデータ送信が完了した時点でコールバックが呼び出されます。

接続している相手のAIDのリストは、DWC_GetAIDList関数で取得することができます。

DWC_SendUnreliableBitmap関数を使えば、複数の相手へのUnreliable送信を一度に行うことができます。

引数

aid 送信先のAID
buffer 送信データへのポインタ
size 送信データのサイズ

返り値

TRUE データ送信が受け付けられた。
FALSE データ送信が下記の理由で受け付けられなかった。
  • エラー発生中
  • 無効なAID
  • 最大データサイズ以上のデータを送信しようとした
