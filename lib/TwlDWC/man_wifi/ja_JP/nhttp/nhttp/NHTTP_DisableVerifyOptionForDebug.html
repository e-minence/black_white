NHTTP_DisableVerifyOptionForDebug

構文


#include <nitroWiFi/nhttp.h>

int NHTTP_DisableVerifyOptionForDebug( NHTTPConnectionHandle handle, u32 verifyOption );

引数

handle NHTTP_CreateConnection()で生成した接続ハンドル
verifyOption CPS SSL: 未使用。単に呼び出すだけ証明書の検証は無効になります。
NSSL: 無効に設定する検証内容をあらわす以下のビットの論理和
SSL_VERIFY_COMMON_NAME 証明書のcommonnameがリクエストURLのホスト名と一致するか検証します。
証明書のcommonnameはドメインレベル毎にワイルドカード「*」が使用可能です。
（例 「*.a.com」は「foo.a.com」に合致しますが、「bar.foo.a.com」は合致しません。「f*.com」は「foo.com」に合致しますが、「bar.com」には合致しません。）
SSL_VERIFY_ROOT_CA 証明書をROOTCAで検証します。
SSL_VERIFY_CHAIN 証明書チェインが正しいかを検証します。
SSL_VERIFY_SUBJECT_ALT_NAME RFC2818 に準拠した subjectAltName 拡張領域の dNSName と iPAddress を 使用したサーバ確認を行います。 ただし、現在の実装では複数の dNSName には対応していません。
SSL_VERIFY_DATE 証明書の期限切れを検証します。

返り値

成功すれば0を、接続ハンドルがNULLの場合に失敗し-1を返します。

説明

SSL通信で送信すべきHTTPリクエストに対し、指定したビットのSSL通信の検証内容を削除します。
接続先のサーバはSSLに対応している必要があります。
また、別途SSL用の設定関数を呼び出しておく必要があります。

注意

この関数は開発用に用意されているものであり、製品に使用することはできません。
製品版の ROM には、この関数がリンクされないように注意してください。
