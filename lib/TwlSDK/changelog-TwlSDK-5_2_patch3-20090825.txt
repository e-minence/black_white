    5_2-patch3-20090825 で修正された内容
    --------------------------------------------------------------------------

    ・(OS) TWL モードでの引数データの扱いについてリファレンスに追記

    OS_GetArgc 関数、OS_GetArgv 関数で取得できる引数データは、FINALROM モード
    では無効となり、それぞれ 0, NULL となるのが仕様ですが、TWL-SDK 5.2 では
    TWL モードでは FINALROM でも引数データが有効となり DEBUG, RELEASE モード同
    様に値を取得することが出来ました。
    本件について OS_GetArgc 関数, OS_GetArgv 関数, デバッグ(引数)(overview) の
    ドキュメントに追記しました。


    ・(OS) TWL 専用 ROM を NITRO で動作した場合の警告表示に関する修正

      TWL 専用 ROM を NITRO で動作させた場合、これまでは OS_Init 関数内で判定
      して警告を表示していました。
      しかし C++ 等では NitroStartUp 関数で OS_Init 関数を呼び出す場合がありま
      すが、 TWL 専用 ROM の NITRO 動作時にはこの NitroStartUp 関数をスキップ
      するようにしていたため警告が表示されませんでした。
      そこで OS_Init 関数で判定するのではなく、_start 関数で判定するようにして
      OS_Init 関数の呼び出し場所に関わらず警告を表示できるように修正しました。


    5_2-patch1-20090617 で修正された内容
    --------------------------------------------------------------------------

    ・(TP) ペンアップを正しく認識できない不具合の修正

      CODEC-TWL モードでタッチパネルを使用した場合、個体差により稀にペンアップ
      を正しく認識することができず、結果的にペンアップの瞬間に不正な座標データ
      が取得される場合がありました。
      この問題を修正するために CODEC のパラメータ調整を行いました。
