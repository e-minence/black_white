CARD_IdentifyBackup

構文

#include <nitro/card.h>

BOOL CARD_IdentifyBackup(CARDBackupType type);

引数

type CARDに搭載されているバックアップデバイスの種類
CARD_BACKUP_TYPE_NOT_USEを指定することはできません

返り値

デバイスへの1バイトリードが正しく成功すれば TRUE を返します。カードが抜けているか接触不良の場合には FALSE を返します。

説明

CARD に搭載されているバックアップデバイスを指定します。
デバイスの指定とともに先頭1バイトの読み出し処理を試行し、成功すれば TRUE を、そうでないなら FALSE を返します。
成否に関するCARDResult型の詳細な結果値はCARD_GetResultCode( )で取得することができます。

注意

  • この関数による指定以降、CARD 関数は該当デバイスが確かに搭載されている前提のもとに動作します。
    実際に搭載されていないデバイスを指定してアクセスすることはプログラムの異常動作および搭載デバイスの破損につながりますので、絶対に避けてください。
  • この関数の返り値はカードの接触不良を確認することはできますが、搭載デバイスが正しい種類かどうかの確認はできません。 たとえ返り値がTRUEであっても、実際に搭載されているデバイスが異なる場合がありえるという点に注意してください。
  • この関数は内部でPXIを介してARM7プロセッサと通信しています。 PXIの割り込みが有効でないと処理が完了しない点に注意してください。
  • この関数を使用する場合、事前にアクセスバスをCARD_LockBackup( )でロックしている必要があります。
